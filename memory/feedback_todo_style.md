---
name: TODO completion style
description: Use checkboxes for completed TODO items; always update TODO.md when committing
type: feedback
---

Use `- [x]` checkboxes to mark TODO items as done. Do not use strikethrough (~~...~~).

Always update TODO.md to reflect completed work before committing — include it in the same commit as the code changes, not a separate follow-up commit.

**Why:** User preference for readability and keeping TODO in sync with the codebase.

**How to apply:** Before every commit, update TODO.md checkboxes and descriptions to reflect what was just done. Stage TODO.md together with the code changes.
